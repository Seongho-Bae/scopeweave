# ScopeWeave Planner User Guide

## 개요

ScopeWeave Planner는 프로젝트용 WBS를 순수 HTML/CSS/JavaScript만으로 관리하는 정적 웹 애플리케이션입니다.

## 주요 사용 흐름

1. 상단에서 **프로젝트 이름**과 **기준일**을 입력합니다.
2. 하단 **최상위 작업 추가** 또는 각 행의 **＋** 버튼으로 작업을 추가합니다.
3. 행 클릭 또는 **✎** 버튼으로 인라인 편집 모드를 엽니다.
4. 계획/실적 날짜와 실적진척상태를 입력하면 요약 수치와 가중치가 자동 재계산됩니다.
5. **CSV 내보내기** / **CSV 가져오기** / **간트차트보기**로 산출물을 활용합니다.

## 계층 규칙

- 최대 깊이는 3단계입니다.
- 1단계: `단계`
- 2단계: `Activity`
- 3단계: `Task`
- 드래그앤드롭은 같은 부모를 가진 같은 계층끼리만 허용됩니다.
- 드래그 시에는 대상 작업과 하위 작업이 함께 이동합니다.

## 저장 방식

### 기본 저장

- 모든 변경은 즉시 브라우저 `localStorage`에 저장됩니다.

### `wbs.json` 자동저장 연결

- Chromium 계열 브라우저에서 **wbs.json 자동저장 연결** 버튼을 누르면 쓰기 가능한 `wbs.json` 파일을 연결할 수 있습니다.
- 연결이 끝나면 변경할 때마다 같은 JSON 스키마로 자동 저장됩니다.
- 외부 저장 JSON에는 내부 계층 관리용 synthetic row / internal id 필드가 포함되지 않습니다.

## CSV 사용

- 내보내기 파일명: `wbs_export_YYYYMMDD.csv`
- 가져오기는 동일한 컬럼 계약을 가진 CSV를 기준으로 전체 데이터를 교체합니다.

## 검증 규칙

- 날짜는 반드시 `YYYY-MM-DD` 형식이어야 합니다.
- 계획종료일은 계획시작일 이상이어야 합니다.
- 실적종료일은 실적시작일 이상이어야 합니다.
- 종료일이 시작일보다 빠르면 표 안에 빨간 경고가 표시됩니다.

## 간트 차트

- 계획 최소 시작일 ~ 최대 종료일 범위를 기준으로 주차 헤더를 생성합니다.
- 주차 안에는 월요일~금요일 날짜가 표시됩니다.
- 계획 바는 진회색(`#333333`), 실적 바는 초록(`#34cb03`)으로 덮어씌워집니다.

## 정적 호스팅 주의사항

- GitHub Pages 같은 정적 호스트에서는 서버 파일을 직접 덮어쓸 수 없습니다.
- 따라서 기본 보장은 `localStorage` autosave이며, 파일 자동저장은
  브라우저가 File System Access API를 지원할 때만 가능합니다.
